v3.0-v3.2 建立了 9 个 learning 模块（1463 行），但 token 收支比只有 3/10：

| 问题 | 影响 |

**原则：每行代码必须在 <30 任务内回本。否则砍掉。**

**现状：** `buildMinimalContext()` 已写好但没被调用。execute prepare 仍用全量 adapter。

**改动：** cmd-execute.ts 的 `handleExecute` 中，用 `buildMinimalContext` 替代全量 brain 注入。

```typescript
// 当前（全量注入，10-20K tokens）：
const instruction = adapter.generateInstruction(task);
const brainContext = getBrainContext(projectDir, task.name);
const enrichedInstruction = brainContext ? instruction + '\n\n' + brainContext : instruction;

// 改为（精简注入，1-5K tokens）：
const minimal = buildMinimalContext(projectDir, task, target);
return { taskId, target, instruction: minimal.instruction, estimatedTokens: minimal.estimatedTokens };
```

**预期：** subagent 上下文 -60~80%，立即生效。

**改动：** auto-learn.ts 中 guard 失败时，将 suggestions 写入 task 的 context（下次 execute prepare 自动包含）。

## F4: Brain 读代码内容（P1）

**现状：** evolveFromTask 只看 `task.files` 路径，推断太浅。

**改为：** 读变更文件的前 30 行（通常含模块注释/JSDoc），提取真实描述。

```typescript
// 在 evolveFromTask 中：
for (const file of task.files || []) {
  const fullPath = path.resolve(projectDir, file);
  if (!fs.existsSync(fullPath)) continue;
  // 只读前 30 行，提取模块描述
  const head = fs.readFileSync(fullPath, 'utf8').split('\n').slice(0, 30).join('\n');
  const docComment = head.match(/\/\*\*([\s\S]*?)\*\//)?.[1]?.trim();
  if (docComment) {
    const existing = brain.architecture.keyModules.find(m => m.path === dir);
    if (existing && !existing.role.includes(docComment.slice(0, 50))) {
      existing.role = docComment.slice(0, 80); // 真实描述替代启发式
    }
  }
}
```

**预期：** Brain 的 keyModules.role 从 "API layer" 变为 "Route a task to the appropriate executor based on scoring rules"。质量提升 3x。
