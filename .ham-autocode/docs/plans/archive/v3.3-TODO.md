# ham-autocode v3.3 TODO

> **主题：Token 节约 — 让每次对话消耗更少、持续更久**

---

## 问题分析

当前每次会话 token 消耗过高，主要瓶颈：

| 消耗源 | 每次消耗 | 频率 | 占比 |
|--------|---------|------|------|
| core/index.ts (721行) | ~15K tokens | Agent 频繁 Read | 高 |
| Skill 全文注入 (auto=213行) | 5-8K/次 | 每次调用 skill | 高 |
| PostToolUse hook 无效执行 | ~0.5K | 每次工具调用 | 中 |
| Subagent 重复上下文 | 10-20K/次 | 每个 subagent | 高 |
| types.ts (299行) | ~6K | 频繁读取 | 中 |
| CLAUDE.md (128行) | ~3K | 每条消息 | 低（不可避免） |

**目标：综合节省 40-60% token 消耗。**

---

## T1: 拆分 index.ts 为独立 command 模块（P0）

**问题：** 721 行巨型 switch-case，Agent 每次想了解某个命令都要 Read 全文（~15K tokens）。

**方案：** 拆分为 `core/commands/*.ts`，每个文件只包含一个 command group。

```
core/commands/
  pipeline.ts    # pipeline init/status/log/pause/resume/mark-interrupted
  dag.ts         # dag init/next-wave/complete/fail/retry/skip/status/visualize/...
  context.ts     # context prepare/budget/summary/search
  route.ts       # route <id>/batch/confirm
  validate.ts    # validate detect/<id>
  recover.ts     # recover checkpoint/rollback/worktree-*
  execute.ts     # execute prepare
  learn.ts       # learn analyze/suggest/apply/patterns/...
  health.ts      # health check/quick/drift/uncommitted/esm-cjs
  research.ts    # research init/report/status
  quota.ts       # quota status/mark-*
  misc.ts        # config/trace/session/commit/teams/rules/spec/token
```

**index.ts 缩减为：** ~50 行的 dispatcher（读 command 名 → 动态 import 对应模块）。

**预期节省：** Agent Read 从 15K → 2-3K tokens（只读需要的 command 文件）。

### 任务
- [ ] T1.1: 创建 `core/commands/` 目录，按 command group 拆分
- [ ] T1.2: index.ts 改为轻量 dispatcher（动态路由到 command 模块）
- [ ] T1.3: 确保 8/8 测试通过

## T2: Skill 分阶段加载（P0）

**问题：** `/ham-autocode:auto` 有 6 个阶段（detect → init → plan → execute → review → ship），但每次调用都注入全部 213 行。实际只需要当前阶段的 20-30 行。

**方案：** 拆分 auto skill 为分阶段子 skill。

```
skills/auto/SKILL.md          → 精简入口（~30行），只描述整体流程 + 阶段跳转
skills/auto/phase-1-detect.md → Phase 1 检测
skills/auto/phase-2-init.md   → Phase 2 初始化
skills/auto/phase-3-plan.md   → Phase 3 规划
skills/auto/phase-4-execute.md→ Phase 4 执行
skills/auto/phase-5-review.md → Phase 5 审查
skills/auto/phase-6-ship.md   → Phase 6 发布
```

**入口 SKILL.md 逻辑：** 检测当前 pipeline 阶段 → 只加载对应 phase 文件。

**预期节省：** 5-8K → 1-2K tokens/次（减少 70%）。

### 任务
- [ ] T2.1: 拆分 auto SKILL.md 为 6 个 phase 文件
- [ ] T2.2: 入口 SKILL.md 改为阶段检测 + 跳转
- [ ] T2.3: 其他大 skill（detect, ship）同样精简

## T3: PostToolUse hook 快速退出（P1）

**问题：** 每次工具调用都执行 `node dist/index.js context budget`，99% 情况下 budget 正常，输出空。白白消耗 ~0.5K tokens + 执行时间。

**方案：** 在 shell 层面快速判断，避免启动 Node 进程。

```bash
# 快速退出：如果没有 pipeline，直接退出
if [ ! -f "$PROJECT_DIR/.ham-autocode/pipeline.json" ]; then
    exit 0
fi

# 快速退出：如果 budget 文件不存在或 consumed < threshold，直接退出
# 用简单的文本检查替代完整 Node CLI 调用
```

**预期节省：** 99% 的 PostToolUse 调用零开销。

### 任务
- [ ] T3.1: 重写 on-post-tool-use.sh — 添加快速退出路径
- [ ] T3.2: 只在 budget 文件存在且 consumed > advisory 阈值时才启动 Node

## T4: Subagent 精简上下文模板（P1）

**问题：** 每个 subagent spawn 时传递完整项目背景（10-20K tokens），很多信息对当前任务无关。

**方案：** 为不同 agent 类型定义最小上下文模板。

```typescript
// core/executor/context-template.ts

interface AgentContext {
  task: { id, name, spec, files };     // 必需：任务信息
  projectHint: string;                  // 精简：project brain 的 1-2 行摘要
  conventions: string;                  // 精简：编码约定（命名/导入风格）
  relatedEntities: string[];           // 精简：只传相关代码实体
}

// codex: 只需 task + files + conventions（~2K tokens）
// opencode: 只需 task + files（~1K tokens）
// claude-code: task + brain context + entities（~5K tokens）
```

**预期节省：** subagent 上下文从 10-20K → 1-5K tokens（减少 60-80%）。

### 任务
- [ ] T4.1: 创建 `core/executor/context-template.ts` — 按 target 生成最小上下文
- [ ] T4.2: 修改各 adapter (claude-code/codex/opencode) 使用模板
- [ ] T4.3: execute prepare 输出精简后的 instruction

## T5: SessionStart hook 精简（P2）

**问题：** SessionStart 执行 3 次 Node CLI 调用（pipeline status + dag status + context budget），拼接输出。

**方案：** 合并为单次 CLI 调用。

```bash
# 当前：3 次 node 调用
node "$CORE_CLI" pipeline status
node "$CORE_CLI" dag status
node "$CORE_CLI" context budget

# 优化：1 次调用返回全部
node "$CORE_CLI" session summary  # 已有，复用
```

**预期节省：** hook 执行时间减少 60%，输出更精简。

### 任务
- [ ] T5.1: 新增 `session context` CLI — 返回精简的 pipeline+dag+budget 摘要
- [ ] T5.2: 重写 on-session-start.sh 使用单次调用

## T6: types.ts 按需导出（P2）

**问题：** 299 行全部类型定义在一个文件，Agent 每次都读全文。

**方案：** 拆分为领域文件，types.ts 只做 re-export。

```
core/types/
  pipeline.ts   # PipelineState, PipelineStatus, PhaseState, LogEntry
  task.ts       # TaskState, TaskStatus, TaskSpec, TaskScores, ...
  config.ts     # HarnessConfig, ContextConfig, ValidationConfig, ...
  routing.ts    # RoutingTarget, RoutingDecision, ...
  recovery.ts   # RecoveryStrategy, CheckpointResult, ...
core/types.ts   # export * from './types/pipeline.js' ...（~20行）
```

**预期节省：** Agent 只读需要的类型文件（~1-2K vs 6K）。
